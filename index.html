SECTION 01

What is a Text Embedding?

Computers can't read words — they only understand numbers. An embedding is a list of decimal numbers (a vector) that represents the meaning of a word or sentence. Similar meanings → similar numbers → close together in space.

🗺️
The GPS Analogy

Just like GPS coordinates (40.71°N, 74.00°W) pinpoint New York on a map, a text embedding like [0.32, -0.14, 0.87, ...] pinpoints "cat" in meaning space. Cities close on a map → similar coordinates. Words with similar meanings → similar vectors.

🎨
The Colour Analogy

Any colour can be described as 3 numbers: RGB (255, 0, 0) = red. Similarly, any word can be described as hundreds of numbers. "King" and "Queen" would have similar numbers, except for certain dimensions that encode gender.

💡
The Big Idea: After training on billions of sentences, a model learns that "cat" and "kitten" should have similar vectors, "hot" and "cold" should be opposite, and King − Man + Woman ≈ Queen (the famous Word2Vec arithmetic!).

SECTION 04

How Are Vectors Actually Learned?

We didn't hand-craft the numbers — a neural network learned them by reading billions of sentences and adjusting values until similar words had similar vectors.

1
Massive Text Corpus (input)
The model is fed billions of sentences: Wikipedia, books, websites. Each word co-occurring with similar contexts starts getting nudged toward similar vectors.
string "The cat sat on the mat. The dog sat by the door."
2
Tokenization + Embedding Lookup
Text → tokens → integer IDs → look up each ID's row in the embedding matrix (a big table of random numbers, initially). The matrix starts as noise.
E ∈ ℝ^(V × d)
Embedding matrix: V = vocab size (30,000), d = dimensions (384)
3
Transformer Layers (the neural network)
The BERT/MiniLM architecture has layers of "attention" — each token looks at all other tokens and decides which ones are relevant. The article's MiniLM model has 6 layers × 12 attention heads.
4
Loss Function → Backpropagation
The model predicts a masked word, compares to the real word, measures error (loss), then propagates gradients back through all layers — adjusting the embedding matrix so similar words get closer vectors.
Δw = −η · ∇_w L
Weight update: η = learning rate, ∇L = gradient of loss
5
Result: Meaningful Vector Space
After training, the embedding matrix contains vectors where direction = meaning. cat and dog are close; cat and algorithm are far apart. This is the matrix used at inference time.
SECTION 05

The Math Behind It

Three operations power text embeddings: the dot product, the L2 norm, and cosine similarity. Let's understand each one.

① Dot Product
A · B = Σ aᵢ × bᵢ
= a₁b₁ + a₂b₂ + … + aₙbₙ
What it means: Multiply matching elements of two vectors, then sum them all up. A scalar (single number) result.

Example: A=[1, 2, 3], B=[4, 5, 6]
A·B = (1×4) + (2×5) + (3×6) = 4 + 10 + 18 = 32
② L2 Norm (Magnitude)
‖v‖ = √(Σ vᵢ²)
= √(v₁² + v₂² + … + vₙ²)
What it means: The "length" of a vector — Pythagoras theorem in N dimensions.

Example: v = [3, 4]
‖v‖ = √(3² + 4²) = √(9 + 16) = √25 = 5.0

Why? We divide by this to make ‖v̂‖ = 1 (unit vector).
③ Cosine Similarity — The Key Formula
cos(θ) = (A · B) / (‖A‖ × ‖B‖)
What it measures: The angle θ between two vectors.
NOT how long the vectors are — only their direction matters.

This means "I love cats" and "I really really love cats a lot" get the same similarity score as "I love cats" vs "I love cats" — because they point in the same direction!
cos(θ) value Meaning
+1.0 Identical direction → same meaning
+0.5 Related (60° apart)
0.0 Unrelated (90° apart)
−1.0 Opposite meaning (180° apart)

SECTION 08

Mean Pooling — Explained

After the transformer produces a vector for each token, we need one vector for the whole sentence. Mean pooling averages them.

📐 The Formula

v_sent = (1/N) × Σ v_tᵢ
v_sent = sentence embedding vector
N = number of tokens
v_tᵢ = embedding vector for token i

Step-by-step for 3 tokens, dim=2:
Token 1: [0.2, 0.8]
Token 2: [0.6, 0.4]
Token 3: [0.4, 0.6]
─────────────────
Sum: [1.2, 1.8]
÷ 3: [0.4, 0.6] ← pooled vector

❓ Why Not Just Use [CLS] Token?

BERT-style models have a special [CLS] token at the start. In fine-tuned models (like for classification), you'd use [CLS]. But for sentence embeddings, mean pooling across all tokens usually works better because:

  • It captures information from every word
  • It's more robust to sentence length
  • [CLS] requires fine-tuning to be meaningful
  • MiniLM (the article's model) is trained specifically for mean pooling

SECTION 09

The Complete Pipeline

Here's everything together — the exact same steps the article's Rust/ONNX code runs, but explained in plain English.

🔤
Step 1: Tokenize string → string[]
Split raw text into tokens. Real models use WordPiece or BPE (Byte-Pair Encoding) to handle unknown words by splitting them into sub-words. "unbelievable" → ["un", "##believ", "##able"]
🔢
Step 2: Token IDs + Attention Mask string[] → int64[]
Look up each token in the vocabulary table. Also create an attention mask (array of 1s and 0s) that tells the model which positions are real tokens vs padding. The article's code creates input_ids, attention_mask, and token_type_ids as three separate arrays.
🧠
Step 3: ONNX Model Inference int64[] → float32[][]
Pass the integer arrays into the MiniLM ONNX model. It runs all 6 transformer layers and outputs a float32 tensor of shape [1, N, 384] — one 384-dim vector per token. The article uses ONNX Runtime for maximum speed.
🏊
Step 4: Mean Pooling float32[][] → float32[]
Average all N token vectors element-wise into a single [384] vector. The article iterates over all token/dimension pairs and divides by seq_len.
✅
Step 5: L2 Normalize float32[] → unit float32[]
Compute the L2 norm, then divide every element by it. Result: a unit vector with ‖v‖ = 1.0. Now you can compute cosine similarity as a plain dot product. The article returns this as Vec<f32> — a Rust vector of 32-bit floats.
🦀
In the Rust article: the embedd() function (note: typo in original!) does exactly these 5 steps. The output is Vec<f32> — a heap-allocated array of 32-bit floats — which gets serialized to JSON and sent back as the API response.
SECTION 10

What Can You Build With This?

Text embeddings are the foundation of most modern NLP applications.

🔍
Semantic Search
Find documents with similar meaning, not just matching keywords. Stack Overflow, GitHub Copilot, and Notion AI all use this.
🤖
RAG (Chat with Docs)
Retrieve relevant document chunks via similarity, then feed them to an LLM. ChatGPT plugins, Claude's context, all use retrieval via embeddings.
🏷️
Text Classification
Spam detection, sentiment analysis, topic labeling — train a simple classifier on top of embeddings.
📌
Recommendation
"Users who liked X also liked Y" — if two product descriptions have high cosine similarity, they're related.
🌐
Multilingual Matching
Multilingual models map "cat" (English) and "gato" (Spanish) to similar vectors — allowing cross-language search.
🧩
Deduplication
Find near-duplicate content in large datasets. If cos(θ) > 0.95, the texts are essentially the same.
